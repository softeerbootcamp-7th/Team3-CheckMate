# 📁 프로젝트 구조 및 레이어 정의

## 1. Root 디렉토리 (src)

| 폴더/파일명 | 정의 및 설명 |
| --- | --- |
| **components** | 컴포넌트를 관리하는 폴더 |
| **pages** | 각 페이지를 정의하며, 페이지당 하나의 디렉토리로 관리 |
| **hooks** | 커스텀 훅을 관리하는 폴더 |
| **utils** | 훅이 아닌 일반 유틸리티 함수 |
| **services** | API 요청 관련 로직 |
| **types** | 각종 타입 (Types) 정의 |
| **constants** | 상수 정의 |
| **stores** | 전역 상태 관리 정의 |
| **routes** | Route 정의 (Flat 하게 관리) |
| **main.tsx** | 애플리케이션 진입점 |
| **global.css** | Tailwind CSS 및 전역 CSS 설정 |

## 2. 레이어 하위 디렉토리 정의 규칙

### 💡 기본 원칙

1. **도메인 단위 구성**: **pages**, **hooks**, **utils**, **types**, **constants**, **stores** 등 대부분의 레이어는 기본적으로 도메인(페이지) 단위로 나누어 관리합니다.
2. **공통 요소 (shared)**: 도메인과 상관없이 재사용되는 요소는 각 레이어 하위의 **shared** 폴더에서 관리합니다.
3. **도메인 깊이**: 도메인 하위 디렉토리는 최대 **1 depth** 까지 만 허용합니다.
4. **높은 응집도 관리**: 드롭다운, 모달과 같이 관심사가 같고 응집도가 매우 높은 컴포넌트는 하나의 폴더 안에서 관리합니다.
5. **배럴 파일 사용**: 폴더 내의 구성 요소를 외부에서 참조할 때 일관성을 유지하고 가독성을 높이기 위해 항상 **index.ts** 배럴 파일 을 사용합니다.

## 3. 서비스 도메인 정의

우리 프로젝트에서 사용하는 주요 서비스 도메인 영역은 다음과 같습니다.

| 도메인 명 | 설명 |
| --- | --- |
| **auth** | 로그인 관련 도메인 |
| **onboarding** | 계정 생성 후, 사업자명 등록 관련 도메인 |
| **dashboard** | 매출, 메뉴, 날씨 분석 등의 지표를 한눈에 볼 수 있는 대시보드 페이지 관련 도메인 |
| **sales** | 매출 분석 도메인 |
| **menu** | 메뉴 분석 도메인 |
| **weather** | 날씨 분석 도메인 |
| **daily-report** | 하루 리포트 관련 도메인 |
| **setting** | 설정 관련 도메인 |
| **ingredient** | 식재료 분석 관련 도메인 |
| **ai-chat** | AI 사이드 챗 관련 도메인 |

## 4. 상세 구조 예시

### components (컴포넌트 레이어)

- `/shared`: 공통 컴포넌트
- `/a도메인`: 특정 도메인 컴포넌트
    - `/a1 도메인`: 하위 도메인 (최대 1 depth)
    - `/shared`: a도메인 내 하위 도메인 간 공용 컴포넌트

```
/components
  ├── /a도메인
  │   ├── /a1 도메인
  │   │   └── /dropdown
  │   │       ├── index.ts
  │   │       ├── dropdown.tsx
  │   │       ├── ... (기타 관련 파일)
  │   └── /shared
  │       └── index.ts
  └── /shared
      └── /공통 컴포넌트
          ├── index.ts
          └── ooo.tsx

```

### 도메인 기반 레이어 (pages, hooks, utils, types, constants, stores)

- 위 레이어들은 모두 동일하게 도메인별 디렉토리 구조를 가집니다.
- 범용적으로 사용되는 요소는 각 레이어 루트의 **shared** 폴더에서 관리합니다.

```
/hooks (또는 utils, types, constants, stores, pages)
  ├── /a도메인
  │   ├── /a1 도메인
  │   │   ├── index.ts
  │   │   └── 파일명.ts
  │   └── index.ts
  ├── /b도메인
  └── /shared
      └── index.ts

```

### services (서비스 레이어)

- API 요청 로직을 관리하며, 파일명을 통해 메서드를 명시적으로 분리합니다.

```
/services
  ├── /a도메인
  │   ├── index.ts
  │   ├── keys.ts
  │   ├── options.ts
  │   ├── get.ts
  │   ├── post.ts
  │   ├── delete.ts
  │   └── put.ts
  ├── /a1 도메인
  └── /shared
      └── index.ts

```
