# 1. 프론트엔드 코드 컨벤션

<body>
<h2>1. 기술 스택 및 기본 규칙</h2>
<h3>1.1 핵심 기술</h3>
<ul>
<li><strong>Framework</strong>: React (함수형 컴포넌트 사용)</li>
<li><strong>Styling</strong>: Tailwind CSS</li>
<li><strong>UI Library</strong>: Shadcn UI
<ul>
<li>모든 shadcn/ui 컴포넌트는 <code>@/components/shared/shadcn-ui</code> 경로에서 관리하고 import 해야 합니다.</li>
</ul>
</li>
</ul>
<h3>1.2 타입 정의 (TypeScript)</h3>
<ul>
<li>객체 데이터 구조나 Props 를 정의할 때는 <strong>interface</strong> 를 사용합니다.</li>
<li>객체 정의 시 <strong>type</strong> 보다는 <strong>interface</strong> 사용을 우선합니다.</li>
</ul>
<h2>2. 컴포넌트 및 네이밍 규칙</h2>
<h3>2.1 컴포넌트 선언</h3>
<ul>
<li>모든 함수형 컴포넌트는 <strong>Arrow Function</strong> 을 사용하여 선언합니다.</li>
<li><strong>Exception</strong>: 내부 shadcn/ui 컴포넌트의 경우 표준 함수 선언(function declaration) 형식을 허용합니다.</li>
</ul>
<h3>2.2 네이밍 컨벤션</h3>

대상 | 규칙 | 예시
-- | -- | --
Folder | kebab-case | user-profile, auth-form
File (.tsx) | PascalCase | UserProfile.tsx
TS/Hook/Util | camelCase | apiClient.ts, useAuth.ts
Hook Name | use 접두어 + camelCase | useFetchData
Constant | UPPER_SNAKE_CASE | API_BASE_URL


<h2>3. 프로젝트 구조 및 레이어링 (src)</h2>
<h3>3.1 디렉토리 구성</h3>
<p><code>src</code> 폴더 하위는 다음과 같은 레이어로 구분하며, 도메인 중심(Domain-Driven)으로 조직합니다.</p>
<ul>
<li><code>components</code>, <code>pages</code>, <code>hooks</code>, <code>utils</code>, <code>services</code>, <code>types</code>, <code>constants</code>, <code>stores</code>, <code>routes</code></li>
</ul>
<h3>3.2 구조적 제약</h3>
<ul>
<li>도메인별로 레이어를 구성하며, 하위 디렉토리의 깊이(Depth)는 최대 <strong>1</strong> 로 제한합니다.</li>
<li>모든 폴더는 클린 내보내기를 위해 반드시 <strong>index.ts</strong> (Barrel File) 파일을 포함해야 합니다.</li>
</ul>
<h2>4. 임포트 규칙 (Import Rules)</h2>
<h3>4.1 경로 설정</h3>
<ul>
<li><strong>Same Domain</strong>: 동일 도메인 내의 구성 요소를 참조할 때는 <strong>Relative Path</strong> 를 사용합니다. (예: <code>./SubComponent</code>)</li>
<li><strong>Different Domains</strong>: 다른 도메인을 참조할 때는 도메인 레벨까지 명시된 <strong>Absolute Path</strong> 를 사용합니다. (예: <code>@/components/auth</code>)</li>
<li><strong>Shadcn UI</strong>: 반드시 <code>@/components/shared/shadcn-ui</code> 경로를 통해 <strong>Absolute Path</strong> 로 import 합니다.</li>
</ul>
<h3>4.2 타입 임포트</h3>
<ul>
<li>TypeScript 타입을 가져올 때는 반드시 <code>type</code> 키워드를 명시해야 합니다.
<ul>
<li>예: <code>import type { UserInfo } from '@/types/user';</code></li>
</ul>
</li>
</ul>
<h2>5. 서비스 도메인 정의</h2>
<p>프로젝트에서 유효한 도메인 범위는 다음과 같습니다. 이외의 도메인 생성 시 팀과 논의가 필요합니다.</p>
<ul>
<li><code>auth</code>, <code>onboarding</code>, <code>dashboard</code>, <code>sales</code>, <code>menu</code>, <code>weather</code>, <code>daily-report</code>, <code>setting</code>, <code>ingredient</code>, <code>ai-chat</code></li>
</ul>
