```markdown
## AdminPage UI 구현 계획 (단일 파일: `frontend/src/pages/admin-page/AdminPage.tsx`)

### 1. 요약
- 목표: 제공된 이미지와 최대한 유사한 관리자 페이지 UI를 `AdminPage.tsx` 한 파일에서 구현합니다.
- 범위: **기본 클릭 상태 포함** (메뉴 선택, 토글 선택, 총 결제금액 반영).
- 반응형: **데스크톱 고정 우선** (작은 화면에서 가로 스크롤 허용).
- 스타일: `index.css` 의 토큰(`text-grey-*`, `bg-grey-*`, `rounded-*`, `body-*` 등)과 shadcn/ui 컴포넌트를 적극 사용합니다.
- 구조화 준비: 향후 컴포넌트 분리 용이하도록 섹션 경계마다 한국어 주석을 배치합니다.

### 2. 파일/구조 설계 (`AdminPage.tsx` 내부)
- 단일 파일 내부에 아래 순서로 배치합니다.
1. `import` 구문
2. 화면 렌더링용 상수 데이터 (메뉴 목록, 판매 유형/주문 수단/결제 수단 옵션)
3. 로컬 `interface` 선언 (객체/props 형태는 `type` 대신 `interface`)
4. `AdminPage` 컴포넌트 (Arrow Function)
5. 파일 내부 보조 렌더 함수 또는 작은 내부 컴포넌트 (필요 시)

- 주석 배치 계획:
1. `// ===== Header 영역 =====`
2. `// ===== 좌측 메뉴 그리드 영역 =====`
3. `// ===== 우측 주문서 상단(빈 상태/선택 상태) =====`
4. `// ===== 우측 주문 옵션 토글 영역 =====`
5. `// ===== 우측 결제 요약/주문 버튼 영역 =====`

### 3. UI 상세 구현 계획
1. 페이지 루트
- 전체 높이: `h-full` + 내부 최소 높이 보장(`min-h-screen` 계열)로 화면 점유.
- 카드형 외곽 래퍼: `bg-grey-0`, `rounded-400`, `border`, `overflow-hidden`.
- 데스크톱 기준 2열 레이아웃:
  - 좌측: 메뉴 영역 (넓은 영역)
  - 우측: 주문서 영역 (고정 폭)

2. 상단 헤더
- 좌측: 서비스명/관리자 페이지 텍스트.
- 우측: 유저 이메일, 매장 연동 상태, POS 연동 상태 텍스트.
- 이미지 톤에 맞춘 구분선 및 여백 적용.

3. 좌측 메뉴 그리드
- 스크롤 가능한 영역(`overflow-y-auto`).
- 메뉴 카드 15개 더미 렌더 (이름/가격).
- 카드 클릭 시 선택 상태 표시:
  - 배경/테두리/텍스트 강조.
- 성능/안정성:
  - 고정 `id` 기반 `key` 사용.
  - 가격 계산은 `useMemo` 로 파생값 관리.

4. 우측 주문서 패널
- 상단 제목 + 주문건수 배지.
- 메뉴 미선택 시 빈 상태(아이콘 + 안내 문구).
- 메뉴 선택 시 선택된 항목 요약(이름/가격) 표시.

5. 주문 옵션 토글(Shadcn)
- `@/components/shared/shadcn-ui` 에서 `ToggleGroup`, `ToggleGroupItem`, `Button` 사용.
- 섹션:
  - 판매 유형 (예: 홀/포장/배달)
  - 주문 수단 (예: POS/키오스크/배달앱)
  - 결제 수단 (예: 카드/현금/간편결제/기타)
- 각 그룹은 `type="single"` + 로컬 상태 연동.
- 선택값이 없을 때도 UI 깨지지 않게 기본값 지정.

6. 결제 요약
- 선택 메뉴 가격을 총 결제금액으로 표시.
- 주문 버튼은 이미지 스타일에 맞춰 비활성 톤 기본 적용, 최소 클릭 가능 상태 유지.

### 4. 인터페이스/타입 변경 사항
- 외부 API 변경 없음.
- `AdminPage.tsx` 내부에 다음 `interface` 추가:
1. 메뉴 아이템 구조 (`id`, `name`, `price`)
2. 토글 옵션 구조 (`value`, `label`, 선택적 `icon`)
- 타입 import 필요 시 `import type` 형식만 사용.

### 5. 테스트/검증 시나리오
1. 초기 진입 시
- 헤더/좌측 메뉴/우측 주문서 레이아웃이 이미지와 유사하게 렌더링되는지 확인.
- 우측 빈 상태 문구가 노출되는지 확인.

2. 메뉴 선택 시
- 클릭한 카드가 활성 스타일로 변하는지 확인.
- 우측 주문서에 선택 메뉴 정보가 표시되는지 확인.
- 총 결제금액이 해당 메뉴 가격으로 반영되는지 확인.

3. 토글 선택 시
- 각 그룹에서 단일 선택만 유지되는지 확인.
- 그룹 간 상태가 독립적으로 유지되는지 확인.

4. 작은 화면 시
- 데스크톱 고정 우선 정책대로 가로 스크롤이 가능하고 레이아웃이 깨지지 않는지 확인.

5. 정적 검증
- `npm run lint`
- `npm run ts:check`

### 6. 가정 및 기본값
1. 이번 구현은 `AdminPage.tsx` 단일 파일만 수정합니다.
2. 실제 주문 API/데이터 연동 없이 더미 데이터 기반 UI 상태만 구현합니다.
3. 아이콘은 프로젝트 내 사용 중인 `lucide-react` 를 사용합니다.
4. 디자인 시스템 우선순위는 `index.css` 토큰/유틸 클래스 > shadcn 기본 스타일 오버라이드 순서로 적용합니다.

```
