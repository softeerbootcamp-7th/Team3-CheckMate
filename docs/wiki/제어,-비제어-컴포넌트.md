### 제어, 비제어 컴포넌트

통제의 주체 = ‘react’. react가 통제 할 수 있냐 없냐

- 제어
    
    React에 의해 값이 제어되는(랜더링이 발생하는) 컴포넌트
    
    - react의 state를 통해 입력값 저장
    - 입력값(state)의 변경에 따른 즉각적인 렌더링이 발생한다.
        
        -> 사용자가 입력을 하는 액션이 발생할 때 마다 리렌더링 발생 
        
    - 유효성 검사를 실시간으로 가능
        
        → 사용가 글자 입력하는 동시에 조건(ex: 8글자 이상) 검증해 사용자가 실시간으로 유효성 위반 확인 가능
        
        → 조건부로 제출 버튼 비활성화 가능
        
        → 실시간으로 입력 형식 적용하기
        
    - `<input value={state} onChange={...} />` 형태.
- 비제어
    - **DOM 자체에서 입력값을 추적**하며, **주로 ref**를 사용하여 입력값에 접근
    - 입력 변경 자체로는 리렌더링이 발생하지 않음
    - 제출/필요 시점에만 `ref.current.value`로 값을 읽는다.
    - 유효성 검사는 값을 읽을때(submit 등)에만 가능
        
        → 제출하는 시점에 조건(ex: 8글자 이상) 검증해서 사용자가 제출 버튼 누른 후에 유효성 위반 확인 가능
        
    - `<input defaultValue="..." ref={...} />` 형태.

⇒  `React Hook Form` 는 비제어 컴포넌트를 사용해 렌더링을 최적화하는 라이브러리. **필요하다면 특정 필드만 Controlled**로 바꿀 수 있다.

### 컴포넌트(필드)를 폼에 등록하는 방법

- 네이티브 DOM 요소, 비제어 컴포넌트 → register() 사용
- 제어 컴포넌트 → <Controller /> 레퍼 컴포넌트 사용

### 컨트롤러 컴포넌트

react-hook-form은 **비제어 컴포넌트와 네이티브** input 사용을 기본 철학으로 함. 근데 외부 제어 컴포넌트(Controlled Component)를 사용하는 상황을 완전히 피하기는 어려움. 그럴 때 `Controller` 를 사용해 외부 제어 컴포넌트를 React Hook Form과 연동할 수 있음.

( 단 제어 컴포넌트가 비제어 컴포넌트가 되는건 아님. 그냥 연동가능하게 해줌)

= 외부 제어 컴포넌트를 React Hook Form에서 사용하고 싶을 때 사용하는 래퍼 컴포넌트
