<p>코멘트 예시 ex) P1 : 코멘트</p>

중요도 | 명칭 | 의미 및 액션
-- | -- | --
P1 | Must Fix | 반드시 수정해야 함. 버그가 있거나 장애 발생 가능성이 높을 때 사용합니다. (승인 거절의 사유)
P2 | Should Fix | 가급적 수정 권장. 당장 큰 문제는 없으나 설계상 결함이 있거나 유지보수가 어려울 때 사용합니다.
P3 | Recommend | 수정하면 좋음. 더 나은 코드 스타일이나 효율적인 로직이 있을 때 제안합니다. (작업자의 선택 존중)
P4 | Question / Nit | 질문 또는 사소한 의견. 코드가 궁금해서 물어보거나, 오타 같은 아주 작은 수정 사항(Nitpick)일 때 사용합니다.
P5 | Praise / FYI | 칭찬 또는 단순 참고. 코드가 깔끔할 때 칭찬하거나, 관련해서 알아두면 좋은 정보를 공유할 때 사용합니다.
